# astrbot_plugin_wuwa_echo

鸣潮（Wuthering Waves）声骸评分 AstrBot 插件。**纯自然语言驱动** —— 发图 + 说一句话，LLM 自动调用评分工具。

## 用法

**触发条件**（两层防护）：

1. **@ 机器人** —— 由 AstrBot 的群聊唤醒机制保证（群聊中不 @ 不会触发；私聊无此要求）
2. **意图关键词** —— 消息文本里要出现下面任一关键词，否则插件拒绝评分：

   > 评分 / 打分 / 评级 / 几分 / 多少分 / 鉴定 / 词条 / 品质 / 怎么样 / 好不好 / 强不强 / echo / 声骸 / 鸣潮 …

   故意避开了生僻字「骸」，常用词都能命中。

### 示例

| 你说（@ bot） | 自动触发 |
|---|---|
| *（发图）+ "今汐这个怎么样？"* | ✅ 单件评分（命中"怎么样"） |
| *（发整套图）+ "维妈这套打个分"* | ✅ 整套评分（命中"打分"，别名→维里奈） |
| *（发图）+ "几分"* | ✅ 评分（命中"几分"） |
| *（发图）+ "啥啊这"* | ❌ 没意图关键词，不触发 |
| *"鸣潮支持哪些角色？"* | ✅ 列角色（命中"鸣潮"） |
| *"风主有哪些别名？"* | ✅ 列别名 |
| *"重载权重"* | ✅ 热重载 |

### 图和艾特不在同一条消息也能用

插件会自动跨消息拼接（5 分钟有效期）：

| 流程 | 行为 |
|---|---|
| **先发图，再 @bot 评分** | 用户先发声骸截图（不 @），插件后台缓存；后续 @bot 说"今汐打分"时自动用缓存图 |
| **先 @bot 再发图** | @bot 说"维妈评分"且没附图 → bot 回"请发图，我会自动续上" → 用户下一条图触发评分（无需重新 @）|
| **缺角色** | 流程中只要任意一条消息带角色名/昵称都能拼上 |

没有 slash 指令——所有交互都通过 LLM 的 function calling 完成。

## 工作原理

插件向 AstrBot 注册了 4 个 LLM 工具（function calls）：

| 工具 | 触发条件 |
|---|---|
| `score_wuwa_echo` | 用户发图 + 提到 "评分/打分/怎么样/品质/词条" |
| `list_wuwa_characters` | 用户问"支持哪些角色/角色列表" |
| `show_wuwa_aliases` | 用户问"X 的别名/昵称" |
| `reload_wuwa_weights` | 用户提"重载/更新配置" |

LLM 看到这些工具描述后自己决定调用谁、传什么参数。例如 *"维妈整套帮我看看"* → LLM 会调用 `score_wuwa_echo(character="维妈", mode="set")`，工具内部把"维妈"解析为「维里奈」再打分。

## 评分算法

双指标制，满分 50：

- **词条种类分（0–25）**：5 条副词条的权重等级之和 / 满潜力（5 × 3 = 15），归一化到 25 分
- **词条数值分（0–25）**：每条副词条 `(当前值 / 满级最大值) × 权重` 加权平均，归一化到 25 分

权重等级语义：

| 等级 | 含义 |
|---|---|
| 3 | 核心（暴击 / 暴击伤害 / 角色专属伤害加成） |
| 2 | 重要（攻击% / 主要技能伤害加成 / 共鸣效率） |
| 1 | 次要（固定攻击 / 副技能伤害加成） |
| 0 | 无用（对该角色） |

评级阈值（可在 `weights.json` 改）：

| 评级 | 分数门槛 |
|------|---------|
| ACE  | ≥ 45    |
| SSS  | ≥ 35    |
| SS   | ≥ 25    |
| S    | ≥ 18    |
| A    | ≥ 10    |
| N    | < 10    |

## 数据来源

- **词条识别**：通过 AstrBot 当前配置的多模态 LLM 识别图片（推荐 Qwen-VL、GPT-4o、Gemini 等支持图片输入的模型）。**未配置多模态模型时插件会报错**，请先到 AstrBot WebUI 选一个 Vision 模型作为当前 provider。
- **角色权重模板**：从 [anyul.cn](https://www.anyul.cn) 抓取的 50 个角色的副词条权重，按 `contribution = weight × max_value` 分桶到 0/1/2/3 等级
- **角色别名**：从 [XutheringWavesUID](https://github.com/loping151/XutheringWavesUID) 的 `char_alias.json` 抓取，含 500+ 条玩家常用昵称

## 昵称模糊匹配

角色名按四级回落解析：

1. **exact** — 原样命中规范名（如 `今汐`）
2. **alias** — 不区分大小写命中任一别名（如 `jx`、`龙女`、`verina`）
3. **substring** — 查询是某规范名/别名的子串，且唯一命中（如 `维里` → `维里奈`）
4. **fuzzy** — `difflib` 相似度匹配（cutoff = 0.6，处理错别字 / 单字简称）

未命中任何角色时回落到 `generic_crit` 通用双暴模板，并在回复里提示。

## 自定义

### 加角色权重

编辑 `data/weights.json` 的 `characters` 段：

```json
"我家角色": {
  "_desc": "自家配置",
  "暴击": 3, "暴击伤害": 3,
  "重击伤害加成": 3,
  "攻击%": 2, "攻击": 1,
  "共鸣技能伤害加成": 2,
  "共鸣解放伤害加成": 1,
  "共鸣效率": 0, "普攻伤害加成": 0,
  "生命%": 0, "防御%": 0, "生命": 0, "防御": 0
}
```

### 加别名

编辑 `data/aliases.json`：

```json
"今汐": ["今汐", "jx", "xx", "汐汐", "龙女", "你给的新昵称"]
```

改完任意文件后对 LLM 说"重载权重"即可生效，无需重启 AstrBot。

## 配置项

`_conf_schema.json` 暴露的可调项：

| Key | 默认 | 说明 |
|---|---|---|
| `default_character` | `generic_crit` | LLM 工具未传角色名时的兜底模板 |
| `show_substat_breakdown` | `true` | 是否在回复里逐条列出副词条得分（2G 内存机器建议关掉） |

## 已知限制

- 识别精度取决于多模态模型质量，小模型可能把「攻击%」和「攻击」搞混
- 主词条目前不参与评分，仅作展示（声骸主词条相对固定，真正决定品质的是副词条）
- 套装合鸣效果不参与评分
- 权重模板按当前版本主流玩法生成，部分边缘流派（如纯辅助治疗 + 攻击塔）可能需要手改
- LLM 必须支持 function calling 才能触发工具；不支持的本地小模型可能无法识别意图
